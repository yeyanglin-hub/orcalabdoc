# 数据采集失败如何排查？

## 问题
在使用OrcaLab进行VR遥操作数据采集时，有时数据采集脚本可能无法正常工作，导致数据无法保存或采集过程中断。我应该如何排查和解决数据采集失败的问题？

## 回答

数据采集失败是一个常见但可能涉及多个环节的问题，这通常与**VR设备连接、OrcaLab客户端状态、数据采集脚本配置、文件写入权限或磁盘空间**等因素有关。系统性的排查是解决问题的关键。

## 📋 数据采集失败排查与解决方案

### 1. **检查Pico VR设备与PC连接**

#### 现象
-   数据采集脚本报错，提示无法连接VR设备或接收VR输入。
-   VR手柄操作没有在仿真中反映。

#### 排查与解决
-   **确保Pico已开机并运行`OrcaGymCtrl`**：检查Pico设备上是否已安装并启动`OrcaGymCtrl`应用程序。
-   **检查ADB连接**：
    -   `adb devices`：确认Pico设备被PC正确识别，状态为`device`。
    -   `adb reverse tcp:8001 tcp:8001`：确保此命令已执行，且每次重启PC或重插Pico后都重新执行。
-   **USB连接**：确保USB数据线连接稳定，未松动。
-   **USB调试授权**：检查Pico设备是否授权了PC进行USB调试。

### 2. **检查OrcaLab客户端状态**

#### 现象
-   数据采集脚本无法连接OrcaLab。
-   OrcaLab客户端卡死或异常退出。

#### 排查与解决
-   **OrcaLab客户端正常启动**：确保OrcaLab客户端以"无仿真程序（手动启动）"模式正常启动，并加载了正确的场景和机器人。
-   **检查客户端日志**：查看OrcaLab客户端"终端栏"是否有错误或警告信息。
-   **重启客户端**：如果客户端状态异常，尝试重启OrcaLab。

### 3. **检查数据采集脚本配置与代码**

#### 现象
-   脚本启动报错，提示`ModuleNotFoundError`、`FileNotFoundError`、`KeyError`等。
-   脚本运行但没有数据输出。

#### 排查与解决
-   **Conda环境激活**：确保在运行脚本前激活了正确的Conda环境：`conda activate orcalab`。
-   **脚本依赖**：确保脚本所需的所有Python包已通过`pip install -r requirements.txt`安装。
-   **配置文件路径与内容**：
    -   检查脚本引用的配置文件（如`example.yaml`）路径是否正确。
    -   检查配置文件中的内容，如`actor`、`task`等模块的参数是否正确，特别是资产路径和站点名称。
    -   确保配置文件中的资产版本与实际订阅的资产版本匹配。
-   **脚本逻辑**：
    -   仔细阅读脚本的报错信息，定位到具体的代码行。
    -   在脚本中添加`print()`语句，输出关键变量，进行调试。
    -   验证VR输入数据是否正确接收，以及机器人控制指令是否正确发送。

### 4. **文件写入权限与磁盘空间**

#### 现象
-   脚本报错，提示`Permission denied`或`No space left on device`。
-   数据采集完成后，找不到保存的文件。

#### 排查与解决
-   **检查输出目录权限**：确保数据采集脚本有权限在指定的输出目录（如`~/OrcaManipulation/data`）创建和写入文件。
    ```bash
    ls -l /path/to/data/directory
    # 如果没有写入权限，尝试
    chmod u+w /path/to/data/directory
    ```
-   **检查磁盘空间**：
    ```bash
    df -h
    ```
    确保有足够的磁盘空间来保存采集的数据，特别是图像和视频数据可能非常大。

### 5. **资产依赖与场景加载**

#### 现象
-   脚本启动后，仿真场景中缺少机器人或目标物体。
-   机器人无法移动或抓取。

#### 排查与解决
-   **资产订阅**：登录OrcaLab资产库网页，确认所有数据采集任务所需的资产包都已成功订阅。
-   **客户端同步**：确保在订阅资产后，重启了OrcaLab客户端，使资产同步到本地。
-   **布局加载**：确保OrcaLab客户端加载了包含机器人和目标物体的正确场景布局。

## 📝 综合排查流程

1.  **从终端报错信息入手**：这是最重要的第一步，报错信息会指引您。
2.  **检查VR连接**：确保Pico设备和PC之间的物理和软件连接都正常。
3.  **检查OrcaLab客户端**：确认客户端正常运行，场景加载无误。
4.  **检查脚本和配置**：仔细核对数据采集脚本的代码和配置文件中的参数。
5.  **检查写入权限和磁盘空间**：确保可以正常保存数据。

## 💡 最佳实践

-   **逐步测试**：在进行复杂的数据采集前，先尝试运行最简单的示例，确保基础功能正常。
-   **详细日志**：在数据采集脚本中启用详细日志输出，记录更多调试信息。
-   **备份**：在尝试进行任何大规模数据采集前，备份您的脚本和配置文件。

## 📝 总结

数据采集失败是一个多因素问题，需要从VR设备连接、OrcaLab客户端、数据采集脚本、文件权限和磁盘空间等方面进行系统性排查。通过仔细检查每个环节，并根据终端报错信息进行调试，可以有效解决数据采集失败的问题。
