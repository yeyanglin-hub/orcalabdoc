# 数据采集脚本如何运行？

## 问题
在OrcaLab中进行VR遥操作数据采集时，我需要运行特定的Python脚本来启动数据记录。我应该如何运行这些数据采集脚本？有哪些关键步骤和注意事项？

## 回答

OrcaLab的VR遥操作数据采集是通过**独立的Python脚本**来启动和管理的。这个脚本负责连接OrcaLab仿真环境、接收VR手柄数据、控制机器人以及记录数据。成功运行数据采集脚本是完成VR遥操作任务的关键一步。

## 📋 数据采集脚本运行流程

### 前提条件

-   **OrcaLab客户端已启动**：OrcaLab客户端应以"无仿真程序（手动启动）"模式运行，并加载了包含目标机器人和场景的布局。
-   **Pico设备已连接并配置**：Pico VR头显已开机，`PicoController.apk`（OrcaGymCtrl）已运行，ADB连接已建立 (`adb reverse`)。
-   **OrcaManipulation代码库已就绪**：`OrcaManipulation`仓库已克隆到本地，且`requirements.txt`中的所有依赖已安装。

### 第一步：激活OrcaLab的Conda环境

-   数据采集脚本通常依赖OrcaLab的Python包和Conda环境中的其他库。因此，在运行脚本前，务必激活正确的Conda环境。
    ```bash
    conda activate orcalab
    ```

### 第二步：进入数据采集脚本目录

-   使用`cd`命令进入包含数据采集脚本的目录。例如：
    ```bash
    cd ~/OrcaManipulation/src/examples/dataCollection
    ```

### 第三步：启动数据采集脚本

-   使用`python`命令运行数据采集脚本。确保您指定了正确的脚本文件名。
    ```bash
    python data_collection_tele.py
    ```
    -   **注意**：在启动命令中，可能还需要根据您的任务需求添加额外的命令行参数。例如，指定配置文件路径、输出目录等。请查阅脚本内部或其`README.md`文件获取详细信息。

### 第四步：观察终端输出

-   脚本启动后，会在终端中实时显示日志信息，包括连接状态、VR数据接收、机器人控制反馈以及数据保存状态等。请密切关注这些输出，以确认脚本是否正常运行。



### 第五步：开始VR遥操作

-   当脚本提示已进入数据采集状态后，戴上Pico VR设备，开始使用手柄进行遥操作。
-   按照任务提示，例如，机械臂复位、抓取目标物体，放置到指定位置等。
-   使用左手柄扳机控制数据采集的开始和保存。

## 💡 脚本中的关键参数与配置

数据采集脚本通常会从配置文件（如YAML文件）中读取任务参数。例如，在`OrcaManipulation`的示例中，可能会有一个`example.yaml`文件：

```yaml
# example.yaml 示例配置
level_name: "shop_pick_and_place"
type: "pick_and_place"
actor:
  names: ["product_A", "product_B"]
  spawnable: ["assets/path/to/product_A", "assets/path/to/product_B"]
# ... 其他随机化、任务逻辑配置
goal:
  name: "MedicineChest"
  site: "MedicineChest_site"
```

### 关键配置项
-   `level_name`：场景名称标识，用于日志记录和数据集区分。
-   `type`：任务类型定义，如`pick_and_place`（抓取放置）。
-   `actor`：动态加载物体的配置，包括名称、路径、关节、随机规则等。
-   `task`：任务目标配置，如目标物体名称、放置位点。

## ⚠️ 注意事项

### 1. **Conda环境**
-   务必确保在运行脚本前激活了正确的Conda环境。

### 2. **脚本路径**
-   `cd`到正确的脚本目录，或者在运行`python`命令时提供完整的脚本路径。

### 3. **配置文件**
-   数据采集脚本通常会引用一个配置文件。确保该配置文件路径正确，且其中的参数设置（如资产版本）与您的环境相符。

### 4. **资产包**
-   确保所有在配置文件和脚本中引用的资产包都已在OrcaLab资产库中订阅，并通过客户端同步到本地。

### 5. **终端状态**
-   数据采集脚本通常会持续运行，直到任务完成或手动终止。请确保终端窗口不会意外关闭。

### 6. **异常情况**
-   如果脚本运行报错，请仔细查看终端的报错信息，通常会提供定位问题的线索。

## 📝 总结

运行数据采集脚本是OrcaLab VR遥操作的核心步骤。在确保OrcaLab客户端和Pico设备已正确配置的前提下，您需要在正确的Conda环境中进入脚本目录，然后使用`python`命令启动脚本。请注意脚本中的配置文件路径和参数，并关注终端输出以确保正常运行。
