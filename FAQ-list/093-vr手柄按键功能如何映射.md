# VR手柄按键功能如何映射？

## 问题
在OrcaLab的VR遥操作模式下，Pico Ultra 4手柄的按键和摇杆如何映射到仿真环境中的机器人控制指令？我应该如何了解或自定义这些映射关系？

## 回答

在OrcaLab的VR遥操作数据采集场景中，Pico Ultra 4手柄的按键和摇杆通常会预设一套功能映射，用于直观地控制仿真环境中的机器人（如机械臂）。理解这些映射关系对于有效进行遥操作至关重要。

## 📋 Pico VR手柄功能映射概述

根据OrcaLab的VR遥操作指南，手柄的基本功能映射如下：

### 1. **机械臂复位与抓取模式切换**
-   **操作**：依次按压**左摇杆**、**右摇杆**。
-   **效果**：
    -   仿真环境中机械臂可移动。
    -   机器人进入抓取操作模式。
-   **作用**：这是进入实际操作前的初始化步骤，确保机械臂处于可控状态。

### 2. **左手手柄功能**
-   **Y键长按**：
    -   **操作**：长按左手手柄的`Y`键。
    -   **功能**：触发"左手抓取"操作，通常用于闭合机械臂夹爪。
-   **X键**：
    -   **操作**：按下左手手柄的`X`键。
    -   **功能**：触发"左手放开"操作，通常用于打开机械臂夹爪。
-   **左手柄扳机 (Trigger)**：
    -   **操作**：按一次左手手柄的扳机键。
    -   **功能**：
        -   首次按压：开始数据采集状态。
        -   再次按压：保存采集的数据，并提示保存目录。如果数据采集失败，则进入下一个任务。

### 3. **右手手柄功能**
-   **B键长按**：
    -   **操作**：长按右手手柄的`B`键。
    -   **功能**：触发"右手抓取"操作，通常用于闭合机械臂夹爪。
-   **A键长按**：
    -   **操作**：长按右手手柄的`A`键。
    -   **功能**：触发"右手放开"操作，通常用于打开机械臂夹爪。



## 💡 映射机制与自定义

### 1. **映射机制**
-   VR手柄的物理输入（按键状态、摇杆位置、姿态数据）通过Pico VR设备传输到PC。
-   PC端的`OrcaManipulation`代码库中的Python脚本会接收这些VR输入，并将其转换为OrcaLab仿真环境可以理解的机器人控制指令（如关节力矩、末端执行器速度、抓取状态等）。

### 2. **自定义映射（通过修改Python脚本）**
-   OrcaLab客户端本身可能不提供GUI界面来自定义VR手柄的按键映射。
-   要自定义映射关系，您需要修改`OrcaManipulation`代码库中负责VR输入处理和机器人控制的**Python脚本**（例如`data_collection_tele.py`或相关的控制模块）。
-   在这些脚本中，您可以找到将特定手柄按键或轴数据与机器人动作（如移动、旋转、抓取、释放）关联起来的逻辑。

#### 示例（伪代码）
```python
# 假设在 OrcaManipulation 的某个控制脚本中
def process_vr_input(vr_state):
    robot_action = {}
    
    # 摇杆控制机械臂末端速度
    robot_action['gripper_velocity_x'] = vr_state.right_joystick.x
    robot_action['gripper_velocity_y'] = vr_state.right_joystick.y
    
    # 左手柄Y键控制左手抓取
    if vr_state.left_y_button_pressed:
        robot_action['left_gripper_state'] = 'close'
    elif vr_state.left_x_button_pressed:
        robot_action['left_gripper_state'] = 'open'
        
    # ...更多映射逻辑
    
    return robot_action
```

## ⚠️ 注意事项

### 1. **脚本依赖**
-   手柄的映射功能完全依赖于`OrcaManipulation`代码库中的数据采集脚本。确保您正在运行正确的脚本。

### 2. **Pico设备连接**
-   确保Pico Ultra 4设备已正确配置（安装APK、开启开发者模式、ADB连接正常），否则手柄输入无法被PC端脚本接收。

### 3. **虚拟环境**
-   确保运行VR遥操作脚本的Python环境已激活，并且所有必要的依赖已安装。

### 4. **调试映射**
-   如果手柄操作没有达到预期效果，您可以：
    -   在Python脚本中打印VR输入数据，检查是否能正确接收到手柄事件。
    -   检查脚本中的映射逻辑是否有误。

### 5. **身体姿态跟踪**
-   VR头显通常也会提供头戴设备本身的姿态跟踪。在高级应用中，这些数据可以用于控制机器人基座移动或作为机器人感知输入。

## 📝 总结

OrcaLab的VR手柄按键功能映射通过`OrcaManipulation`代码库中的Python脚本实现。默认提供了一套用于机械臂复位、抓取、释放以及数据采集开始/结束的映射。用户可以通过修改这些Python脚本来自定义手柄的功能映射，以适应特定的机器人控制或任务需求。
