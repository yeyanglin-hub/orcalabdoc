# 如何管理本地资产存储空间？

## 问题
OrcaLab下载的资产包会占用本地硬盘空间，随着订阅的资产越来越多，我可能会面临存储空间不足的问题。我应该如何有效地管理本地资产的存储空间？

## 回答

有效管理本地资产存储空间是确保OrcaLab流畅运行和系统健康的关键。您可以通过多种策略来优化硬盘使用，包括**清理不必要的资产、手动删除文件和定期维护**。

## 📋 本地资产存储空间管理策略

### 1. **仅订阅必要的资产**

#### 策略
- **按需订阅**：在资产库中，只订阅您当前项目或学习阶段真正需要的资产包。
- **避免盲目订阅**：不要一次性订阅所有资产包，尤其是大型资产包。

#### 操作
- 在资产库网页端，仔细查看资产包的"详情"，了解其内容和大小，再决定是否订阅。

### 2. **取消订阅不再需要的资产包**

#### 策略
- 定期审查您的"个人中心"->"已订阅资产"列表，取消订阅不再使用的资产包。

#### 操作
1. 登录OrcaLab资产库（[https://simassets.orca3d.cn/](https://simassets.orca3d.cn/)）。
2. 进入"个人中心"，找到"已订阅资产"。
3. 点击不再需要的资产包旁的"取消订阅"按钮。
4. **重启OrcaLab客户端**：这将阻止客户端在下次启动时再次同步该资产包，并将其从客户端的资产栏中移除。



### 3. **手动删除本地资产文件（慎重操作）**

取消订阅后，本地硬盘上的资产文件并不会自动删除。如果您需要彻底释放空间，可以手动删除。

#### 策略
- 找到OrcaLab本地资产存储目录，手动删除不再需要的资产包对应的文件。

#### 操作
1.  **定位资产存储路径**：
    -   OrcaLab的本地资产通常存储在用户主目录下的某个隐藏文件夹中（如`~/.orcalab/assets/`），或者在Conda环境的`site-packages`目录下的特定子文件夹中。
    -   您可以在OrcaLab客户端启动时的日志或`关于OrcaLab`信息中查找具体的资产存储路径。
    -   **以`OrcaPlayground`项目为例**：其资产可能存储在项目目录下的特定子目录中。
    ```bash
    # 假设资产存储在 ~/.orcalab/assets/
    ls -lh ~/.orcalab/assets/
    ```
2.  **确认文件归属**：在删除之前，**务必确认您要删除的文件确实属于不再需要的资产包，且没有被其他活跃项目使用**。
3.  **执行删除**：
    ```bash
    # 示例：删除某个资产包目录
    rm -rf ~/.orcalab/assets/your_asset_package_name
    ```

#### ⚠️ **警告：`rm -rf`命令具有极高的风险，请务必确认路径和文件，避免误删重要系统文件或正在使用的项目文件。**

### 4. **清理Conda和Pip缓存**

Conda和Pip会在您的系统上缓存下载的包文件，这些缓存也会占用大量空间。定期清理可以有效释放空间。

#### 操作
- **清理Conda缓存**：
  ```bash
  conda clean --all
  ```
- **清理Pip缓存**：
  ```bash
  pip cache purge
  ```

### 5. **优化硬盘使用**

#### 策略
- **使用大容量SSD**：如果预算允许，优先选择容量更大的SSD，尤其是NVMe SSD，以应对日益增长的资产需求。
- **外部存储**：对于不常用的或归档的仿真数据、录制视频等大型文件，可以将其移动到外部硬盘或网络存储（NAS）上。

#### 操作
- 将OrcaLab的本地资产存储目录设置为一个专门的、容量较大的分区或硬盘。

## 📝 总结

管理本地资产存储空间需要**定期审视订阅、取消不再需要的资产、谨慎手动删除文件**，并辅以**清理缓存**等措施。合理的存储管理不仅能释放宝贵的硬盘空间，还能确保OrcaLab运行环境的整洁和高效。
