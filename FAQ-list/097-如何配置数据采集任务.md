# 如何配置数据采集任务？

## 问题
在OrcaLab中进行VR遥操作数据采集时，我需要针对不同的任务（如抓取放置、二维码扫描）配置数据采集脚本。我应该如何配置这些数据采集任务？有哪些核心配置参数？

## 回答

在OrcaLab中配置数据采集任务主要通过**修改数据采集脚本所引用的配置文件（通常是YAML格式）**来实现。这个配置文件定义了任务类型、场景中的动态物体、光源、任务目标等关键信息，使得您可以灵活地定制不同的数据生成场景。

## 📋 数据采集任务配置文件核心模块

数据采集任务的配置文件（例如`OrcaManipulation`项目中的`example.yaml`）通常包含以下几个核心模块：

### 1. **基础信息 (Basic Information)**

| 字段        | 核心作用     | 关键说明                                         |
|-------------|--------------|--------------------------------------------------|
| `level_name` | 场景名称标识 | 用于日志记录、数据集区分、任务回放。建议使用业务语义名称（如`pharmacy_pick`）。                                                                           |
| `type`      | 任务类型定义 | 决定`task`模块的配置结构。当前支持`pick_and_place`（抓取放置）和`scan_qr`（二维码扫描）。                                                                    |

### 2. **动态物体 (Actor)**

| 字段        | 核心作用         | 关键说明                                         |
|-------------|------------------|--------------------------------------------------|
| `names`     | 动态加载物体的名称 | 数组格式，与`spawnable`中的资产路径一一对应。这些是在任务中可能随机出现的物体。                                                                           |
| `spawnable` | 资产路径         | 指向资产库中Prefab（预制件）的路径。路径错误会导致物体无法生成。                                                                                           |
| `joints`    | 物体根关节/可控关节 | 定义物体模型的关节，用于位姿控制。                                                                                                       |
| `joints_dof` | 关节自由度       | `1`=单自由度，`3`=球形关节，`6`=刚体关节。数量需与`joints`中的定义一致。                                                                             |
| `random.qpos` | 关节位置随机化   | `true`=关节位置随机化，`false`=固定。用于增加数据多样性。                                                                                   |
| `random.nums` | actor生成数量范围 | 数组格式（如`[1, 5]`），定义了每次任务中该类型actor的生成数量范围。                                                                     |
| `random.six_dof.center` | 随机中心点     | 世界坐标系下的随机中心点，物体将围绕此点随机生成。                                                                                     |
| `random.six_dof.bound_position` | XYZ方向偏移范围  | 例如`[[-0.1, 0.1], [-0.1, 0.1], [0, 0]]`表示X、Y方向随机偏移±0.1，Z方向固定。                                                                |
| `random.six_dof.bound_rotation` | 绕XYZ轴旋转范围  | 弧度表示，例如`[[0, 0], [0, 0], [0, 1]]`表示绕Z轴随机旋转一个单位弧度。                                                                          |

### 3. **场景灯光 (Light)**

| 字段        | 核心作用     | 关键说明                                         |
|-------------|--------------|--------------------------------------------------|
| `names`     | 灯光实例名称 | 数组格式，定义场景中的灯光实例。                                                                                                   |
| `spawnable` | 灯光prefab路径 | 指向灯光预制件的路径。                                                                                                     |
| `random.position` | 位置随机化   | `true`/`false`，是否启用灯光位置随机化。                                                                                   |
| `random.rotation` | 旋转随机化   | `true`/`false`，是否启用灯光旋转随机化。                                                                                   |
| `random.center` | 随机中心点   | 灯光随机位置的中心点。                                                                                                     |
| `random.bound_position` | 位置偏移范围 | 灯光在XYZ方向的随机偏移范围。                                                                                              |
| `random.bound_rotation` | 旋转范围     | 灯光在XYZ轴的随机旋转范围（弧度）。                                                                                        |
| `random.nums` | 生成数量范围 | 数组格式，每次生成的灯光数量。                                                                                             |
| `random.cycle` | 更新周期     | 每隔`X`个任务更新一次灯光配置，用于增加数据多样性。                                                                        |

### 4. **任务逻辑 (Task)**

| 字段        | 核心作用     | 关键说明                                         |
|-------------|--------------|--------------------------------------------------|
| `type`      | 任务类型     | 必须与顶层`type`一致（如`pick_and_place`）。                                                                               |
| `goal.name` | 目标物体/区域名称 | 任务目标（如抓取或放置）的物体或区域的名称。                                                                               |
| `goal.site` | 放置位点     | 场景中定义的放置目标物的位点。如果不存在，任务将失败。                                                                   |

## 🛠️ 新场景配置步骤

1.  **明确任务类型**：首先确定您的数据采集任务属于哪种类型（`pick_and_place`或`scan_qr`），这将决定配置文件的基本结构。
2.  **验证资产路径**：确保`spawnable`字段中引用的所有资产路径都是有效的，且资产已在资产库中订阅并同步到本地。
3.  **配置actor参数**：定义任务中涉及的所有动态物体（如机器人要操作的物品），包括名称、资产路径、关节信息和自由度。
4.  **设置actor随机规则**：配置物体的随机位置、旋转和生成数量，以增加数据的多样性和泛化能力。注意随机范围不要过大，防止物体穿模或掉落。
5.  **配置light参数**：定义场景中的灯光，并可设置灯光的位置、旋转和数量随机化，进一步增加数据多样性。
6.  **配置task目标**：根据任务类型定义具体的任务目标，例如抓取什么物体，放置到哪个位点。
7.  **小规模验证**：在配置完成后，建议先将`random.nums`设置为`[1,1]`（即只生成一个物体），进行小规模测试，验证配置是否正确，然后再扩大随机范围。

## ⚠️ 常见配置错误与排查

### 1. **资产加载失败**
- **错误**：`spawnable`路径错误。
- **排查**：核对资产库Prefab路径，确保路径无拼写错误，且资产已订阅。

### 2. **机器人行为异常**
- **错误**：`joints`与`joints_dof`数量不一致。
- **排查**：检查这两个字段的数组长度，确保一一对应。

### 3. **物体异常行为**
- **错误**：随机范围过大，导致物体穿模或掉落。
- **排查**：缩小`six_dof`的`bound_position`/`bound_rotation`范围。

### 4. **任务无法完成**
- **错误**：`site`名称不存在。
- **排查**：确认场景中存在`goal.site`指定的位点名称，或修正配置。

## 📝 总结

数据采集任务的配置通过修改YAML配置文件完成，它包含了任务的基础信息、动态物体（actor）、场景灯光和任务逻辑。理解每个模块的字段含义，并根据任务需求进行精确配置，是成功进行VR遥操作数据采集的关键。
