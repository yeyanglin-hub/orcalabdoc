# 仿真运行失败的常见原因？

## 问题
OrcaLab在启动仿真程序后，可能会遇到仿真无法正常运行、报错退出或行为异常的情况。仿真运行失败的常见原因有哪些？我应该如何排查和解决这些问题？

## 回答

OrcaLab仿真运行失败的原因复杂多样，涉及**Python环境、仿真程序代码、场景配置、资产依赖、硬件兼容性**等多个方面。系统化的排查方法能帮助您快速定位问题。

## 📋 仿真运行失败的常见原因与排查

### 1. **Python环境或依赖问题**

#### 现象
-   终端报错`ModuleNotFoundError`、`ImportError`。
-   提示某个Python包不存在或版本不匹配。
-   仿真程序启动后立即退出。

#### 排查与解决
-   **检查Conda环境激活状态**：确保您在启动OrcaLab之前已激活正确的Conda环境（`conda activate orcalab`）。
-   **检查Python依赖安装**：
    -   确认仿真程序（如`OrcaPlayground`）的`requirements.txt`中的所有依赖已安装：`pip install -r requirements.txt`。
    -   检查`pip show orca-lab`，确认其核心依赖是否满足。
-   **清理Python缓存**：`pip cache purge`，然后尝试重新安装相关包。

### 2. **仿真程序代码错误**

#### 现象
-   终端显示Python栈追踪（Stack Trace）错误，指向您的仿真脚本文件。
-   提示`SyntaxError`、`TypeError`、`NameError`等Python代码错误。

#### 排查与解决
-   **仔细阅读错误信息**：Python的错误信息通常非常详细，会指明错误类型、发生的文件和行号。
-   **代码调试**：
    -   在OrcaLab客户端的"终端栏"中查看完整日志。
    -   尝试在单独的终端中直接运行您的仿真Python脚本，跳过OrcaLab客户端，以便更好地调试代码：
        ```bash
        conda activate orcalab
        python your_simulation_script.py [args]
        ```
    -   使用Python调试器（如`pdb`）或IDE（如VS Code）进行断点调试。
-   **检查逻辑错误**：如机器人控制逻辑、任务条件判断等。

### 3. **`.orcalab/config.toml`配置错误**

#### 现象
-   仿真程序未出现在OrcaLab启动对话框中。
-   选择仿真程序后，OrcaLab报错或闪退。
-   提示`Program not found`或`Invalid arguments`。

#### 排查与解决
-   **检查TOML语法**：确保`config.toml`文件没有语法错误（例如缺少引号、括号不匹配）。
-   **`name`和`display_name`唯一性**：确保所有`external_programs.programs`中的`name`和`display_name`都是唯一的。
-   **`command`和`args`路径**：
    -   `command`通常是`"python"`。
    -   `args`中的模块路径（`-m your.module`）或脚本路径（`your_script.py`）是否正确，并且相对于`.orcalab/config.toml`所在的目录。
-   **重启客户端**：每次修改`config.toml`后，务必重启OrcaLab客户端。

### 4. **资产依赖缺失或版本不兼容**

#### 现象
-   仿真启动后，场景中缺少机器人模型或关键道具。
-   终端报错`AssetNotFound`、`ResourceLoadError`。
-   物理交互异常，如机器人"塌陷"或"浮空"。

#### 排查与解决
-   **检查资产订阅状态**：登录资产库网页，确认所有仿真所需的资产包都已成功订阅。
-   **重启客户端同步资产**：确保在订阅或更新资产包后，关闭并重新启动了OrcaLab客户端，使资产同步到本地。
-   **检查资产路径**：在仿真脚本中引用的资产路径是否正确。
-   **清理本地资产缓存**：如果怀疑本地资产文件损坏，可以尝试手动删除相关资产文件，然后重启客户端重新下载。

### 5. **场景配置或布局问题**

#### 现象
-   场景加载异常，物体位置混乱。
-   仿真启动后机器人未在预期位置。

#### 排查与解决
-   **检查布局文件**：如果您加载了自定义布局文件，确保该文件没有损坏，且其中引用的资产都存在。
-   **在"无仿真程序"模式下调试**：使用"无仿真程序（手动启动）"模式加载场景，手动检查每个物体的位置、旋转、缩放和物理属性是否正确。
-   **检查机器人配置**：如果机器人行为异常，检查其URDF/USD文件定义是否正确，关节限制、电机参数等是否合理。

### 6. **硬件驱动或系统资源问题**

#### 现象
-   GPU相关错误（`CUDA Error`、`OpenGL Error`）。
-   内存溢出（`OutOfMemory`）。
-   仿真帧率极低，卡顿严重。

#### 排查与解决
-   **显卡驱动**：检查NVIDIA显卡驱动版本是否满足OrcaLab要求并更新到最新。
-   **系统资源**：使用`nvidia-smi`、`htop`等工具监控GPU、CPU、内存使用情况，确保没有达到硬件瓶颈。
-   **关闭无关程序**：关闭其他占用大量资源的应用程序。

### 7. **网络连接问题**

#### 现象
-   仿真启动时，如果需要联网验证或加载某些在线资源，可能会报错。

#### 排查与解决
-   **检查网络连通性**：确保您的计算机能够正常访问互联网。

## 📝 综合排查流程

1.  **从终端错误信息入手**：仔细阅读OrcaLab客户端"终端栏"或运行仿真脚本的终端输出，它通常会给出最直接的线索。
2.  **分层排查**：按照`Python环境` -> `config.toml配置` -> `仿真代码` -> `资产/场景` -> `硬件/系统`的顺序逐层排查。
3.  **简化问题**：尝试运行最简单的仿真示例，如果简单示例正常，说明问题可能在您的自定义配置或代码中。
4.  **寻求帮助**：如果自行解决困难，提供详细的错误信息、日志和您的操作步骤，向OrcaLab社区或技术支持寻求帮助。
