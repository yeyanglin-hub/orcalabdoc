# 什么是仿真程序？如何选择？

## 问题
OrcaLab中的“仿真程序”指的是什么？在启动仿真时，我应该如何选择合适的仿真程序？

## 回答

在OrcaLab中，“仿真程序”是控制仿真场景中机器人行为、物理交互以及数据处理逻辑的外部Python脚本。它定义了仿真的具体任务和运行方式。

## 📋 仿真程序概述

### 定义
- **仿真程序**：一个独立的Python脚本或模块，用于控制OrcaLab仿真环境。
- **作用**：指定仿真目标、机器人行为、传感器数据处理、任务逻辑等。
- **运行方式**：通过OrcaLab客户端的启动对话框选择并执行。

### 配置文件
所有仿真程序都通过项目根目录下的`.orcalab/config.toml`文件进行配置。



## ⚙️ 仿真程序类型

### 1. **无仿真程序（手动启动）**

#### 特点
- 不执行任何外部Python脚本。
- 仿真环境启动后，需要用户手动在GUI界面进行操作。

#### 适用场景
- **场景编辑与搭建**：在没有脚本控制的情况下，自由拖拽、调整场景中的物体。
- **手动调试**：测试单个物体物理行为、观察碰撞效果。
- **交互式探索**：纯粹的3D环境漫游和体验。

#### 优点
- **灵活性高**：完全由用户控制。
- **操作直观**：通过GUI界面进行实时交互。

### 2. **预定义仿真程序**

#### 特点
- OrcaLab或`OrcaPlayground`等示例项目中内置的仿真脚本。
- 通常用于展示OrcaLab的特定功能或提供开箱即用的仿真体验。

#### 常见示例程序
- **`run_sim_loop`**：空循环仿真，通常用于测试环境。
- **`run_character`**：角色仿真，用于人体或通用角色动画。
- **`run_legged_train`**：足式机器人训练，用于强化学习训练。
- **`run_wheeled_chassis`**：轮式底盘仿真，用于车辆动力学研究。
- **`run_xbot_orca`**：XBot机器人仿真，用于特定机器人平台的控制。
- **`run_ackerman`**：四轮底盘小汽车仿真，用于车辆物理特性模拟。


#### 适用场景
- **快速入门**：学习OrcaLab的典型应用。
- **功能演示**：向他人展示OrcaLab的能力。
- **基础测试**：验证环境配置和硬件性能。

#### 优点
- **方便快捷**：无需额外配置即可运行。
- **参考价值**：可作为开发自定义程序的范例。

### 3. **自定义仿真程序**

#### 特点
- 用户根据自己的需求编写的Python脚本。
- 需要在`.orcalab/config.toml`中进行注册才能在OrcaLab客户端中显示和启动。

#### 适用场景
- **科研项目**：实现特定的机器人控制算法、AI训练任务。
- **教学实验**：自定义教学内容，满足课程需求。
- **原型开发**：验证新的机器人系统或交互逻辑。

#### 优点
- **高度定制化**：完全满足个性化需求。
- **功能扩展强**：可集成各种Python库和算法。

## 💡 如何选择合适的仿真程序

选择合适的仿真程序取决于您的目的和当前任务。

### 1. **初次接触OrcaLab**

- **目的**：了解OrcaLab功能，熟悉基本操作。
- **建议**：
  - 先选择**“无仿真程序（手动启动）”**，熟悉界面和手动操作。
  - 尝试运行**`run_ackerman`**（四轮底盘小汽车仿真），体验交互式物理仿真。
  - 运行其他预定义示例，了解不同功能模块。

### 2. **进行场景搭建或编辑**

- **目的**：创建新的仿真场景，调整物体位置、姿态。
- **建议**：选择**“无仿真程序（手动启动）”**，以便自由编辑场景，不受脚本控制影响。

### 3. **验证特定机器人行为或算法**

- **目的**：测试机器人控制算法、强化学习模型等。
- **建议**：
  - 选择**预定义仿真程序**中与您任务最接近的，作为参考或基础。
  - 如果现有程序无法满足，需要**创建自定义仿真程序**，并将其配置到`.orcalab/config.toml`中。

### 4. **VR遥操作数据采集**

- **目的**：通过VR设备控制机械臂进行数据采集。
- **建议**：
  - 首先启动OrcaLab，选择一个带有机械臂的场景（如shop场景）。
  - 在OrcaLab界面选择**“无仿真程序（手动启动）”**启动。
  - **单独在终端中运行数据采集脚本**（例如`python data_collection_tele.py`），该脚本会连接OrcaLab环境。

## 📝 配置文件`.orcalab/config.toml`

### 配置格式
```toml
[[external_programs.programs]]
name = "your_program_name"           # ⚠️ 必填：程序唯一标识符
display_name = "显示名称"             # ⚠️ 必填：在 OrcaLab UI 中显示的名称
command = "python"                    # ⚠️ 必填：执行命令
args = ["-m", "examples.your_module.run_script"]  # ⚠️ 必填：命令行参数列表
description = "程序描述"              # 可选：程序描述信息
```

### 注意事项
- `name` 和 `display_name` 在所有程序中必须唯一。
- `command` 通常是`"python"`。
- `args` 定义了启动Python脚本的模块或路径，以及传递给脚本的参数。

## ⚠️ 常见问题与排查

### Q: 为什么我配置的仿真程序没有显示？
A: 1. 检查`.orcalab/config.toml`文件路径是否正确。
   2. 检查`config.toml`文件语法是否有误。
   3. 确保`name`和`display_name`在所有程序中都是唯一的。
   4. OrcaLab启动时会自动加载该文件，可能需要**重启OrcaLab**客户端。

### Q: 启动仿真后没有反应？
A: 1. 检查`.orcalab/config.toml`中`command`和`args`配置是否正确。
   2. 查看OrcaLab的**终端栏**是否有错误信息输出。
   3. 检查Python环境是否激活，所需的依赖包是否安装。

### Q: `.orcalab/config.toml`文件在哪里？
A: 该文件通常位于您的项目根目录中。如果您使用的是`OrcaPlayground`项目，它会在`OrcaPlayground/.orcalab/config.toml`。如果没有，您可以使用`orcalab --init-config`命令在当前工作目录生成一个。
