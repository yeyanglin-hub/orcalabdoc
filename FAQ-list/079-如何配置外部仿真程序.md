# 如何配置外部仿真程序？

## 问题
OrcaLab支持运行外部的Python仿真程序。我应该如何配置这些外部程序，以便它们能够出现在OrcaLab客户端的启动列表中，并正确执行？

## 回答

OrcaLab通过**`.orcalab/config.toml`配置文件**来管理外部仿真程序。这个TOML格式的文件允许您定义自定义的Python脚本，并将它们集成到OrcaLab的启动流程中。正确配置该文件是运行自定义仿真任务的关键。

## 📋 外部仿真程序配置流程

### 第一步：创建或定位配置文件

-   **配置文件路径**：OrcaLab启动时会查找当前工作目录下的`.orcalab/config.toml`文件。
-   **创建新文件**：如果您的项目目录中没有`.orcalab/config.toml`文件，您可以使用以下命令生成一个基础配置文件：
    ```bash
    conda activate orcalab
    orcalab --init-config
    ```
    这会在当前目录生成一个`.orcalab/`文件夹，并在其中包含`config.toml`文件。

### 第二步：编辑`config.toml`文件

-   使用文本编辑器（如`nano`, `vim`, `VS Code`等）打开`.orcalab/config.toml`文件。

### 第三步：添加新的仿真程序条目

-   在`config.toml`文件中，找到`[[external_programs.programs]]`部分。每个`[[external_programs.programs]]`块定义一个独立的仿真程序。
-   按照以下格式添加您的新程序信息：

```toml
# 示例：添加一个新的仿真程序
[[external_programs.programs]]
name = "my_custom_sim"               # ⚠️ 必填：程序唯一标识符
display_name = "我的自定义仿真程序"   # ⚠️ 必填：在 OrcaLab UI 中显示的名称
command = "python"                    # ⚠️ 必填：执行命令（通常是 "python"）
args = ["-m", "examples.my_module.run_script", "--config", "my_config.yaml"]  # ⚠️ 必填：命令行参数列表
description = "这是一个自定义的机器人仿真脚本" # 可选：程序描述信息
```

#### 参数详细说明

| 参数           | 类型       | 必填 | 说明                                                                                                                                     |
|----------------|------------|------|------------------------------------------------------------------------------------------------------------------------------------------|
| `name`         | 字符串     | ✅ 是 | **程序唯一标识符**。用于OrcaLab内部查找和启动程序。必须与所有已配置程序的`name`和`display_name`都不重复。建议使用小写字母、数字和下划线，如`my_program`。 |
| `display_name` | 字符串     | ✅ 是 | **显示名称**。在OrcaLab启动对话框的UI中显示给用户。必须与所有已配置程序的`name`和`display_name`都不重复。可以使用中文、空格等字符，如`我的程序`。 |
| `command`      | 字符串     | ✅ 是 | **执行命令**。通常是`"python"`，也可以是其他可执行命令（如`"python3"`、`"conda"`等），但最常用的是`python`来运行Python脚本。   |
| `args`         | 字符串数组 | ✅ 是 | **命令行参数列表**。每个参数作为数组的一个元素。例如：<br>- 模块方式：`["-m", "examples.module.run_script"]`<br>- 脚本方式：`["examples/script.py", "--arg1", "value1"]`<br>- 带参数：`["-m", "examples.module.run", "--config", "config.yaml", "--train"]`               |
| `description`  | 字符串     | ❌ 否 | **程序描述**。用于在OrcaLab UI的工具提示中显示，帮助用户了解程序功能。                                                                   |

### 第四步：保存`config.toml`文件

-   保存您对`config.toml`文件所做的修改。

### 第五步：重启OrcaLab客户端

-   为了让新的配置生效，您需要**关闭并重新启动OrcaLab客户端**。
    ```bash
    conda activate orcalab
    orcalab
    ```
-   重新启动后，您的自定义仿真程序应该会出现在OrcaLab启动对话框的"仿真程序"列表中。

## 💡 配置示例

### 示例1：运行Python模块
```toml
[[external_programs.programs]]
name = "my_module_runner"
display_name = "运行我的Python模块"
command = "python"
args = ["-m", "my_project.scripts.main_script", "--mode", "train"]
description = "启动 my_project 下 main_script 模块的训练模式"
```

### 示例2：运行Python脚本文件
```toml
[[external_programs.programs]]
name = "my_script_runner"
display_name = "运行我的Python脚本"
command = "python"
args = ["scripts/test_sim.py", "--iterations", "1000", "--visualize"]
description = "直接运行项目根目录下的 test_sim.py 脚本"
```

## ⚠️ 重要注意事项

### 1. **`name`和`display_name`的唯一性**
- **严格要求**：`name`和`display_name`在所有配置的程序中必须**全局唯一**。如果重复，OrcaLab可能无法正确识别和启动程序。

### 2. **`args`中的路径问题**
- **相对路径**：`args`中的路径通常是相对于`.orcalab/config.toml`文件所在的**项目根目录**。
- **模块方式 (-m)**：如果使用`-m`参数以模块方式运行，确保Python模块路径正确且模块文件在Python的`sys.path`中可被找到。

### 3. **Python环境**
- 配置的`command = "python"`会使用启动OrcaLab客户端时激活的Conda环境中的Python解释器。请确保您的仿真程序所需的所有依赖都已安装在该Conda环境中。

### 4. **调试**
- 如果配置的外部程序无法启动或运行报错，请：
  - 查看OrcaLab客户端的"终端栏"输出信息。
  - 尝试在单独的终端中直接运行`command`和`args`中的命令，进行调试。

## 📝 总结

OrcaLab通过编辑项目根目录下的`.orcalab/config.toml`文件来配置外部仿真程序。您需要为每个程序定义一个唯一的`name`和`display_name`，指定执行`command`（通常是`python`），并提供详细的`args`命令行参数。保存文件并重启OrcaLab后，您的自定义程序即可在启动对话框中选择运行。
