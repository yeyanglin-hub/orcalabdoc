# 采集的数据保存在哪里？

## 问题
通过OrcaLab的VR遥操作模式进行数据采集后，生成的各种数据（如机器人关节数据、传感器数据、VR手柄数据等）会保存在我的计算机的哪个位置？如何找到这些数据？

## 回答

OrcaLab在VR遥操作模式下采集的数据通常由**数据采集脚本**负责保存到您本地计算机的指定目录。这些数据是进行后续AI训练或行为分析的重要基础。

## 📋 数据采集结果的存储位置

### 1. **由数据采集脚本指定**
-   数据采集脚本（如`data_collection_tele.py`）通常会内置逻辑，将采集到的数据保存到预设的目录。
-   该目录通常是脚本**当前工作目录**下的一个子目录，或者通过**命令行参数**或**配置文件**（如`example.yaml`）指定。

### 2. **查看终端输出**
-   当您运行数据采集脚本时，终端会实时显示日志信息。在数据保存成功后，脚本通常会**明确提示数据保存的路径**。



#### 示例终端输出 (伪代码)
```
...
[INFO] Data collection started for episode 1.
[INFO] Saving collected data to: /home/user/OrcaManipulation/data/2024-01-22_15-30-00_shop_pick_and_place/episode_001.hdf5
[INFO] Data collection finished. Press trigger for next episode or 'q' to quit.
...
```

### 3. **检查脚本的配置文件**
-   数据采集任务通常会有一个对应的配置文件（如`example.yaml`），其中可能包含数据保存路径、文件名格式等参数。
-   请打开该配置文件，查找`output_dir`, `data_path`, `save_location`等相关字段。

#### 示例配置文件片段 (来自OrcaManipulation `example.yaml` 伪代码)
```yaml
# ...
output:
  base_dir: "./data"  # 数据保存的基础目录
  format: "hdf5"      # 数据文件格式
  naming_convention: "{timestamp}_{level_name}/{episode}.{format}"
# ...
```

### 4. **检查脚本源代码**
-   如果上述方法都无法确定保存路径，您可以直接查阅数据采集脚本（如`data_collection_tele.py`）的源代码。
-   在代码中搜索`open()`, `write()`, `save()`, `os.path.join`等函数调用，以找到实际的文件写入逻辑和路径。

## 📁 数据存储格式和内容

数据采集脚本通常会以**结构化的文件格式**存储数据，以便后续处理。常见的数据格式包括：

### 1. **HDF5 (.hdf5)**
-   **特点**：适用于存储大型、复杂、异构的科学数据。支持分层存储和高效读写。
-   **内容**：可能包含机器人关节位置/速度/力矩、末端执行器位姿、VR手柄姿态、传感器读数（相机图像、深度图、激光雷达点云）、时间戳、任务状态、奖励等。

### 2. **NumPy (.npy / .npz)**
-   **特点**：用于高效存储和加载NumPy数组，适用于大量数值数据。
-   **内容**：机器人轨迹、传感器序列数据等。

### 3. **JSON (.json)**
-   **特点**：轻量级的数据交换格式，易于人类阅读和机器解析。
-   **内容**：任务配置、元数据、少量结构化数据或日志。

### 4. **图像/视频文件 (.png, .jpg, .mp4, .avi)**
-   **特点**：由相机传感器采集的图像或录制的视频，通常存储在数据目录下的子文件夹中。

## 💡 最佳实践

### 1. **自定义输出路径**
-   在运行数据采集脚本时，尽可能通过命令行参数或修改配置文件来指定一个明确且易于管理的输出目录，例如：
    ```bash
    python data_collection_tele.py --output-dir /home/user/my_robot_data/experiment_A
    ```

### 2. **结构化命名**
-   使用有意义的命名约定来组织您的数据文件，例如`{任务名称}_{日期}_{时间}/{episode_id}.hdf5`。
-   这有助于在海量数据中快速查找和管理。

### 3. **备份重要数据**
-   定期备份您采集到的重要数据，防止硬盘故障或误删。

### 4. **元数据**
-   除了原始数据，确保同时保存与数据相关的元数据，例如：机器人型号、传感器类型、仿真参数、随机种子、任务描述、操作员信息等。这对于数据分析和复现非常关键。

## ⚠️ 注意事项

### 1. **磁盘空间**
-   数据采集，尤其是包含图像和点云数据时，会产生非常大的数据量。确保您的硬盘有足够的空间。

### 2. **文件权限**
-   确保运行数据采集脚本的用户具有在指定输出目录创建和写入文件的权限。

## 📝 总结

OrcaLab的VR遥操作采集的数据通常由Python脚本保存到指定目录，这个目录可以通过终端输出、脚本配置文件或直接查阅脚本源代码来确定。数据通常以HDF5、NumPy、JSON等格式存储。建议自定义输出路径和结构化命名，并定期备份重要数据。
