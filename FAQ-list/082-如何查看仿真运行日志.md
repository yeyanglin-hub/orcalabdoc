# 如何查看仿真运行日志？

## 问题
OrcaLab仿真程序运行过程中，会产生大量的日志信息，包括程序输出、错误信息、警告等。我应该如何在OrcaLab客户端或终端中查看这些仿真运行日志？

## 回答

查看仿真运行日志是排查问题、监控程序状态和理解仿真行为的关键。OrcaLab提供了两种主要方式来查看日志信息：**客户端的"终端栏"**和**运行仿真程序所在的终端**。

## 📋 查看仿真运行日志的两种方式

### 1. **在OrcaLab客户端的"终端栏"中查看（推荐）**

OrcaLab客户端集成了"终端栏"模块，用于实时显示外部仿真程序的输出信息。

#### 步骤
1.  **启动OrcaLab客户端**：确保OrcaLab客户端已启动并进入GUI界面。
2.  **启动仿真程序**：在客户端中选择并启动您的仿真程序。
3.  **导航到"终端栏"**：
    -   在OrcaLab客户端界面的底部（或根据您的布局设置），找到并点击"终端"（Terminal）标签页，进入"终端栏"模块。
4.  **实时查看日志**：
    -   "终端栏"会实时显示仿真程序的所有标准输出（`stdout`）和标准错误（`stderr`）信息。
    -   您可以滚动查看历史日志，观察程序的运行状态、打印信息、警告和任何错误。


#### "终端栏"功能
-   **清空日志**：点击"终端栏"中的"清空"按钮，可以清除当前显示的日志信息。
-   **复制日志**：点击"终端栏"中的"复制"按钮，可以将所有显示的日志内容复制到剪贴板，方便粘贴到文本编辑器或分享给他人。
-   **搜索日志**：某些终端栏可能提供搜索功能，方便您快速定位特定的关键词或错误信息。

### 2. **在运行仿真程序所在的终端中查看**

如果您是在OrcaLab客户端外部，直接在Linux终端中启动仿真Python脚本（例如，VR遥操作数据采集脚本），那么日志信息会直接输出到该终端窗口。

#### 步骤
1.  **打开独立的终端窗口**：用于运行您的仿真Python脚本。
2.  **激活Conda环境**：
    ```bash
    conda activate orcalab
    ```
3.  **运行仿真脚本**：
    ```bash
    python your_simulation_script.py [args]
    ```
4.  **实时查看日志**：
    -   所有脚本的`print()`输出和错误信息都会实时显示在该终端窗口中。
    -   您可以使用终端的滚动功能查看历史记录。

#### 优势与劣势
-   **优势**：在调试Python脚本时，可以直接看到更原始的输出和完整的Python栈追踪，这对于复杂问题排查非常有帮助。
-   **劣势**：不直接集成在OrcaLab客户端内，需要管理多个窗口。

## 💡 日志级别与详细程度

OrcaLab及其底层组件可能会支持不同的日志级别（如`DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL`）。

-   **默认级别**：通常默认显示`INFO`、`WARNING`和`ERROR`级别的日志。
-   **调整日志级别**：在启动仿真程序时，您可以通过命令行参数或环境变量来调整日志的详细程度。
    -   **环境变量示例**：
        ```bash
        ORCA_LOG_LEVEL=DEBUG orcalab # 启动OrcaLab时启用DEBUG级别日志
        ```
    -   **程序参数示例**：
        ```bash
        python your_simulation_script.py --log-level DEBUG
        ```
    -   **优点**：在排查复杂问题时，`DEBUG`级别的日志可以提供更详尽的内部信息。

## ⚠️ 注意事项

### 1. **日志量**
-   在`DEBUG`或`INFO`级别下，日志量会非常大，可能会迅速填满终端缓冲区。如果需要保存完整的日志，建议将终端输出重定向到文件：
    ```bash
    python your_simulation_script.py > simulation_log.txt 2>&1
    ```

### 2. **错误信息分析**
-   当出现错误时，请重点关注`ERROR`或`CRITICAL`级别的日志，以及Python的栈追踪（Stack Trace）。栈追踪会指出错误发生的文件、函数和行号。

### 3. **时间戳**
-   许多日志会包含时间戳，这对于分析问题发生的时间序列非常有用。

## 📝 总结

OrcaLab的仿真运行日志可以通过客户端的"终端栏"进行实时查看，方便用户监控和管理。对于更深入的Python脚本调试，直接在独立的终端中运行仿真脚本并查看输出会更有效。通过调整日志级别，可以获取不同详细程度的信息，以辅助问题排查。
