# 首次启动OrcaLab为什么会自动安装依赖？

## 问题
OrcaLab在首次启动时，为什么会出现"正在同步资产包..."并自动安装Python依赖包和下载资产？这个过程是做什么的？

## 回答

OrcaLab在首次启动时执行自动安装依赖和同步资产包的操作，是为了确保软件环境的完整性和最新性，为用户提供一个**开箱即用**的仿真体验。

## 📋 自动安装依赖

### 1. **目的**
- **确保Python环境完整性**：OrcaLab作为一个Python应用程序，可能依赖一些在`pip install orca-lab`时未完全安装或动态加载的包。
- **动态库和插件**：可能需要安装一些与OrcaLab核心功能紧密相关的Python模块、插件或绑定，这些在标准`pip`安装中可能不会自动完成。
- **版本匹配**：确保所有依赖包的版本与OrcaLab客户端核心程序兼容。

### 2. **触发机制**
- OrcaLab客户端在首次运行时，会检测其运行环境。如果发现缺少必要的依赖，或某些依赖版本不符合要求，就会自动触发安装过程。

### 3. **处理流程**
- **检查依赖**：扫描当前Python环境中OrcaLab所需的模块和版本。
- **下载与安装**：通过内部的包管理机制，从PyPI或预配置的源下载并安装缺失或需要更新的Python包。
- **更新环境变量**：可能需要更新一些Python路径或环境变量，以确保新安装的依赖可以被正确引用。

### 4. **注意事项**
- **网络连接**：此过程需要稳定的网络连接，以便下载依赖包。
- **权限**：确保当前用户有权限在Conda环境中安装新的Python包。
- **耗时**：根据网络速度和所需依赖的数量，此过程可能需要几分钟到十几分钟不等。

## 📦 自动同步资产包

### 1. **目的**
- **提供开箱即用资源**：确保用户在启动OrcaLab后，能立即获得必要的3D模型、场景、机器人等资产，无需手动下载。
- **保持资产最新**：自动同步最新版本的订阅资产包，确保用户使用的资产是最新的，包含最新的修复或功能。
- **个性化资产**：下载用户在资产库中订阅的自定义资产包。

### 2. **触发机制**
- **首次启动**：OrcaLab客户端启动时，会检查本地资产库与用户订阅信息。如果本地没有或版本过旧，则会触发同步。
- **每次启动**：每次启动OrcaLab时，都会与在线资产库进行一次同步检查。如果检测到新的订阅或现有订阅有更新，就会自动触发下载。
- **资产库订阅更新**：用户在资产库网页中订阅或取消订阅资产后，下次启动OrcaLab时会触发同步。

### 3. **处理流程**
- **连接资产库**：OrcaLab客户端连接到在线资产库服务。
- **身份验证**：使用用户的登录凭据进行身份验证。
- **对比清单**：对比本地资产列表与在线订阅列表和版本信息。
- **下载/更新**：下载新的资产包，并更新已订阅但版本过旧的资产包。
- **本地解压/安装**：将下载的资产包解压到OrcaLab的本地资产目录。



### 4. **注意事项**
- **网络连接**：资产包通常较大，需要良好的网络环境。网络不稳定可能导致下载失败或中断。
- **磁盘空间**：确保本地有足够的磁盘空间来存储下载的资产包。
- **下载速度**：如果资产包下载速度慢，可能需要检查网络或尝试更换网络环境。
- **强制重启**：如果在资产库网页订阅新资产后，OrcaLab需要**关闭并重新启动**才能触发资产下载同步。

## 📝 总结

OrcaLab的首次启动自动安装依赖和同步资产包是一个**智能化的设计**，旨在为用户提供便捷、最新的使用体验。这个过程是确保软件**正常运行**和**拥有所需资源**的关键一步。如果在此过程中遇到问题，通常需要检查网络连接、磁盘空间以及用户权限。
