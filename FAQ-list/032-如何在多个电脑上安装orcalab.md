# 如何在多个电脑上安装OrcaLab？

## 问题
我有多台电脑（例如一台台式机和一台笔记本），我希望在不同的电脑上都能使用OrcaLab。我应该如何操作才能在多个设备上安装和同步OrcaLab？

## 回答

您可以在多台符合系统和硬件要求的电脑上分别安装OrcaLab。由于OrcaLab是单用户桌面应用，所以需要在每台电脑上独立安装。关键在于如何**同步您的项目文件和资产**。

## 📋 多设备安装步骤

### 1. **在每台电脑上独立安装OrcaLab**

- **环境准备**：确保每台电脑都满足OrcaLab的系统要求和硬件配置（尤其是NVIDIA RTX显卡）。
- **操作系统**：在每台电脑上安装Ubuntu 22.04 LTS 或 24.04 LTS。
- **Miniconda**：分别在每台电脑上安装Miniconda。
- **Conda环境**：为OrcaLab创建独立的Conda环境。
- **安装OrcaLab**：执行`pip install orca-lab`安装客户端。
- **启动与资产同步**：首次启动时等待资产包同步完成。

### 2. **同步项目文件和代码**

对于您自己创建的仿真项目（如`.orcalab/config.toml`、布局文件、Python脚本、自定义资产等），强烈推荐使用**版本控制系统**进行同步。

#### 推荐方式：Git + GitHub/GitLab
- **原理**：将您的OrcaLab项目目录初始化为Git仓库，并推送到远程仓库（如GitHub、GitLab）。在另一台电脑上克隆该仓库。
- **操作步骤**：
  1. **初始化Git仓库**（在您的项目目录）：
     ```bash
     cd /path/to/your/OrcaLab_Projects/MySimulationProject
     git init
     git add .
     git commit -m "Initial commit of OrcaLab project"
     ```
  2. **关联远程仓库**：
     ```bash
     git remote add origin https://github.com/your_username/MySimulationProject.git
     git push -u origin master
     ```
  3. **在另一台电脑上克隆仓库**：
     ```bash
     git clone https://github.com/your_username/MySimulationProject.git
     cd MySimulationProject
     ```
  4. **同步更新**：
     - 在一台电脑上工作后，提交并推送到远程：`git add . && git commit -m "Update feature X" && git push`
     - 在另一台电脑上工作前，先拉取最新代码：`git pull`
- **优势**：
  - **版本历史**：清晰的项目版本管理。
  - **冲突解决**：Git提供强大的冲突解决机制。
  - **数据安全**：远程仓库作为备份，防止本地数据丢失。

#### 替代方式：云存储/文件同步工具
- **原理**：使用云存储服务（如Google Drive, Dropbox, OneDrive）或同步工具（如Syncthing）自动同步项目文件。
- **操作**：
  1. 将OrcaLab项目目录放在云存储的同步文件夹中。
  2. 确保在两台电脑上都安装并配置好相应的客户端进行同步。
- **优势**：简单易用，非技术用户也能快速上手。
- **劣势**：
  - **无版本历史**：难以回溯文件历史，误删难以恢复。
  - **冲突处理**：同步工具对文件冲突处理能力较弱，可能需要手动解决。
  - **大文件同步慢**：对于大型资产文件，同步可能耗时较长。

### 3. **资产包同步**

OrcaLab的资产包是与您的用户账号绑定的。您在资产库中订阅的资产，会在您**登录OrcaLab客户端**并在其**启动时自动下载和同步**。

- **原理**：OrcaLab客户端启动时会检测您账号下订阅的所有资产，并自动下载到本地。
- **操作**：
  1. 在每台电脑上启动OrcaLab客户端，并使用**同一个账号登录**。
  2. 等待客户端完成"正在同步资产包..."的过程。
- **优势**：无需手动复制大型资产文件，OrcaLab会自动管理。
- **注意事项**：
  - **网络带宽**：资产包通常较大，首次同步可能需要较长时间和带宽。
  - **磁盘空间**：确保每台电脑都有足够的磁盘空间存储资产包。
  - **同步触发**：订阅新资产或资产更新时，需要关闭并重新启动OrcaLab客户端。

## 💡 最佳实践

### 1. **统一账号**
- 在所有设备上使用**同一个OrcaLab账号**，以便同步订阅的资产。

### 2. **Git管理项目代码**
- 将自定义的Python脚本、配置文件（`.orcalab/config.toml`）、场景布局文件等小型文本文件使用Git进行版本管理和同步。

### 3. **区分数据存储**
- 原始仿真数据（如VR采集的数据）通常比较大，不适合通过Git同步。可以考虑单独存储在云存储、外部硬盘或NAS上。

### 4. **避免同时操作**
- 尽量避免在两台电脑上同时修改同一个项目文件，这容易导致版本冲突。
- 养成"先拉取，再修改，后提交"的习惯。

### 5. **系统环境一致性**
- 尽量保持不同电脑上Ubuntu系统版本、Conda环境、Python版本和已安装依赖的一致性，可以减少兼容性问题。

## 📝 总结

在多台电脑上安装和使用OrcaLab是可行的，关键在于利用**Git进行项目代码同步**和依靠**OrcaLab的自动资产同步机制**来管理资源。通过合理的策略，您可以高效地在不同设备间切换工作。
