# 如何启动仿真？

## 问题
在OrcaLab中如何启动仿真程序？有哪些启动方式和注意事项？

## 回答

在OrcaLab中启动仿真有多种方式，可以根据不同需求选择合适的启动模式。以下是详细的启动流程和操作指南。

## 🚀 基本启动流程

### 第一步：准备仿真环境
```bash
# 1. 确保OrcaLab正在运行
conda activate orcalab
orcalab

# 2. 确保已加载适当的场景和布局
选择场景 → 加载布局 → 添加必要资产
```

### 第二步：启动仿真程序
```
在OrcaLab界面中：
1. 点击右上角绿色【仿真运行】按钮 [▶]
2. 在弹出的对话框中选择仿真程序
3. 点击【启动】开始仿真
4. 观察终端输出确认启动状态
```


## 🎮 仿真程序选择

### 📋 程序类型

#### 1. **无仿真程序（手动启动）**
```
用途：手动控制仿真过程
适用场景：
├── 场景编辑和调试
├── 手动测试物理效果
├── 交互式场景验证
└── 自定义控制脚本

操作方式：
├── 通过界面工具移动物体
├── 使用快捷键控制
├── 鼠标交互操作
└── 实时参数调整
```



#### 2. **预定义仿真程序**
```
常见程序类型：
├── run_sim_loop     - 基础仿真循环
├── run_character    - 角色仿真
├── run_legged_train - 足式机器人训练
├── run_wheeled_chassis - 轮式底盘仿真
├── run_xbot_orca    - XBot机器人仿真
├── run_ackerman     - 四轮底盘小汽车仿真
└── 自定义程序      - 用户配置的程序
```

#### 3. **示例程序启动**
以四轮底盘小汽车为例：
```
1. 添加hummer_h2_usda_1资产到场景
2. 点击【仿真运行】按钮
3. 选择"run_ackerman"程序
4. 启动后使用WASD键控制车辆移动
```



## ⚙️ 仿真程序配置

### 📝 配置文件说明
仿真程序通过`.orcalab/config.toml`文件进行配置：

```toml
[[external_programs.programs]]
name = "run_ackerman"
display_name = "四轮底盘小汽车仿真"
command = "python"
args = ["-m", "examples.wheeled_chassis.run_ackerman"]
description = "控制四轮底盘车辆进行物理仿真"
```

### 🔧 配置参数说明
```
必需参数：
├── name         - 程序唯一标识符
├── display_name - 界面显示名称
├── command      - 执行命令（通常是python）
└── args         - 命令行参数数组

可选参数：
└── description  - 程序功能描述
```

### ➕ 添加自定义程序
```toml
# 示例：添加新的仿真程序
[[external_programs.programs]]
name = "my_custom_sim"
display_name = "我的自定义仿真"
command = "python"
args = ["-m", "my_project.run_simulation", "--config", "config.yaml"]
description = "自定义机器人仿真程序"
```

关键信息类型：

重要指标：

## 🛑 停止仿真

### 正常停止
```
方法1：界面停止
点击【停止仿真】按钮 [⏹]

方法2：程序自然结束
等待仿真程序自动完成

方法3：快捷键停止
使用程序定义的停止快捷键
```

### 强制停止
```
紧急情况处理：
1. 关闭终端窗口
2. 结束Python进程
3. 重启OrcaLab客户端
4. 检查系统资源状态

命令行操作：
# 查找并结束Python仿真进程
ps aux | grep python
kill -9 <进程ID>
```

## ⚠️ 常见启动问题

### 🚨 启动失败排查
```
检查清单：
├── 确认环境配置正确
├── 验证依赖包是否安装
├── 检查配置文件语法
├── 确认资产包已下载
└── 查看详细错误信息

常见错误：
├── 模块导入失败 → 检查Python路径
├── 资产加载失败 → 确认资产包存在
├── 端口占用冲突 → 重启服务或换端口
└── 权限不足错误 → 检查文件权限
```

### 🔧 性能优化建议
```
提升启动速度：
├── 使用SSD存储资产
├── 关闭不必要的后台程序
├── 优化网络连接
└── 更新显卡驱动

提升运行性能：
├── 降低场景复杂度
├── 调整渲染质量
├── 优化物理参数
└── 监控系统资源
```

## 🔄 高级启动技巧

### 批处理启动
```bash
# 创建启动脚本
#!/bin/bash
conda activate orcalab
cd /path/to/project
orcalab &
sleep 10
python -m my_simulation.run_auto
```

### 远程启动
```bash
# SSH远程启动（需要X11转发）
ssh -X user@remote_host
export DISPLAY=:0.0
conda activate orcalab
orcalab
```

### 调试模式启动
```bash
# 启用详细日志
ORCA_LOG_LEVEL=DEBUG orcalab

# 启用性能分析
ORCA_PROFILE=1 python -m simulation.run_program
```
