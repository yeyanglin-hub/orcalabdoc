# 如何停止正在运行的仿真？

## 问题
在OrcaLab中启动仿真程序后，我应该如何安全、正确地停止正在运行的仿真？有哪些停止方法和注意事项？

## 回答

在OrcaLab中停止正在运行的仿真，确保资源被正确释放并且避免数据损坏是很重要的。OrcaLab提供了多种停止仿真程序的方法，您可以根据具体情况选择最合适的方式。

## 📋 停止正在运行的仿真流程

### 1. **通过OrcaLab客户端界面停止（推荐）**

这是停止仿真最安全和推荐的方法，确保OrcaLab能够妥善地清理资源。

#### 步骤
1.  **点击"停止仿真运行"按钮**：在OrcaLab客户端界面的顶部菜单栏右侧，找到并点击**红色的"停止仿真运行"按钮 [⏹]**。
    -   该按钮通常在"仿真运行"按钮 [▶] 的旁边。


#### 效果
-   OrcaLab会向正在运行的外部仿真程序发送停止信号。
-   仿真程序会执行其预定义的清理逻辑（如保存数据、关闭连接）。
-   3D视图中的仿真活动将停止，物理引擎将暂停。
-   "终端栏"可能会显示程序正常退出的日志信息。

### 2. **通过仿真程序内部逻辑停止（高级）**

对于自定义的仿真程序，您可以编写代码实现程序内部的停止逻辑。

#### 方法
-   **监听外部信号**：仿真程序可以监听操作系统的终止信号（如`SIGTERM`），并在收到信号时执行清理工作。
-   **键盘事件**：程序可以监听特定的键盘按键（如`Esc`键），按下时触发程序退出。
-   **达到任务目标**：当仿真程序完成其预设的任务目标时，可以自行退出。

### 3. **通过终端强制终止（不推荐，仅作应急处理）**

如果OrcaLab客户端或仿真程序卡死，无法通过界面正常停止，您可以强制终止进程。**请注意，这种方法可能导致数据未保存或资源未正常释放。**

#### 步骤
1.  **定位仿真程序进程**：在Linux终端中，使用`ps aux | grep python`命令查找正在运行的Python进程，寻找与您的仿真程序相关的进程ID（PID）。
    ```bash
    ps aux | grep python
    # 示例输出：
    # your_user   1234  0.5  0.1  123456 7890 ?        Sl   10:00   0:05 python -m examples.wheeled_chassis.run_ackerman
    # 这里的PID是1234
    ```
2.  **强制终止进程**：使用`kill -9`命令强制终止该进程。
    ```bash
    kill -9 1234 # 将1234替换为实际的PID
    ```
3.  **关闭OrcaLab客户端**：如果OrcaLab客户端本身也卡死，可能需要强制关闭其窗口或通过系统任务管理器结束其进程。

#### 风险
-   数据丢失或损坏。
-   资源（如显存、文件句柄）未正常释放，可能影响后续仿真。
-   可能导致Python环境或OrcaLab客户端状态异常，需要重启或清理缓存。

## ⚠️ 注意事项

### 1. **优先使用界面按钮**
- 始终优先通过OrcaLab客户端界面上的"停止仿真运行"按钮来结束仿真，以确保安全关闭和资源清理。

### 2. **保存数据**
- 如果您的仿真程序在运行过程中会生成数据，请确保在程序停止前，所有重要数据都已保存。通过客户端按钮停止时，程序通常有机会执行保存操作。

### 3. **检查终端日志**
- 停止仿真后，查看OrcaLab客户端的"终端栏"或您运行脚本的终端，确认程序是否正常退出，是否有错误或警告信息。

### 4. **异常退出后的处理**
- 如果发生强制终止或异常退出，建议重启OrcaLab客户端，并检查系统资源（如`nvidia-smi`），确保没有残留进程或显存未释放。

## 📝 总结

安全、正确地停止OrcaLab仿真程序，应优先使用客户端界面上的"停止仿真运行"按钮。在极端情况下，如果程序卡死，可以考虑通过终端强制终止，但需注意可能带来的数据和资源问题。对于自定义仿真程序，也可以通过代码内部实现优雅的退出逻辑。
